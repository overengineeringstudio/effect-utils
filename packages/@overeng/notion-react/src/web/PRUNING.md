# Pruning notes for `vendored-notion.css`

`vendored-notion.css` is a derivative of react-notion-x's `styles.css`
(see `LICENSE-NOTICE.md` for origin and license). The original stylesheet
covers the full Notion page-app surface; we only render a subset of blocks,
so we prune unused sections and rescope global tokens.

Re-vendoring process:

1. Download the upstream `styles.css` for the target version (see
   `LICENSE-NOTICE.md` for the pinned version and commit).
2. Run `tmp/rnx-prune/prune.mjs` (see repo history for the script or
   reconstruct from the list below) against the raw file to emit the
   derivative, or apply the same rules by hand.
3. Update the version and SHA in `LICENSE-NOTICE.md`.
4. Visually re-check all block stories in Storybook.

## Rescope

- Upstream declares design tokens on `:root`, leaking them into any document
  that imports the stylesheet. We rescope the token block so it applies only
  inside `.notion-page` (which is the top-level wrapper emitted by our web
  renderer). Consumers importing `vendored-notion.css` without a
  `.notion-page` wrapper will see no tokens ŌĆö intentional.

## Dropped rule groups

Any CSS rule whose selector matches one of these patterns is dropped:

- `notion-app`, `notion-frame` ŌĆö outer scroll/app chrome (we render inline).
- `notion-nav-*`, `notion-header`, `notion-topbar`, `notion-breadcrumb*` ŌĆö
  top navigation.
- `notion-aside*` ŌĆö right-side table of contents panel.
- `notion-search*` ŌĆö search overlay.
- `notion-comments`, `notion-comment`, `notion-discussion*` ŌĆö inline
  comments.
- `notion-board*`, `notion-gallery*`, `notion-list-view*`,
  `notion-list-collection`, `notion-list-item*`, `notion-list-body`,
  `notion-table-view*`, `notion-table-th`, `notion-table-header*`,
  `notion-table-body`, `notion-table-cell-*`, `notion-collection*`,
  `notion-property-*` ŌĆö collection views and database chrome. We only
  render `simple_table` blocks, not collection views.
- `notion-page-title`, `notion-page-icon-inline`, `notion-page-link`,
  `notion-page-mention` ŌĆö page-header title cluster and mentions
  (we render `child_page`, not page frames).
- `notion-small-text`, `notion-full-width`, `notion-semantic-string`,
  `notion-tweet`, `notion-lite`, `notion-preview-card`, `notion-link-mention*` ŌĆö
  optional chrome not covered by our block set.
- `notion-block-<id>` ŌĆö ID-scoped rules generated by upstream for dynamic
  block formatting; we do not surface block IDs as class names.
- `notion-pdf` ŌĆö upstream PDF viewer chrome; our PDF block is a plain
  download link.

Kept: typography (text, h1ŌĆōh4, quote, callout, code, equation), lists
(to-do, bulleted, numbered, toggle), media (image, video, audio, file,
asset wrapper, page-cover), simple_table, column/column-list, bookmark,
hr, link-to-page, color palette, design tokens.

## When to re-prune

- On upstream release with visual improvements to block-level rendering.
- On upstream release that changes token names (check if our overrides
  in `styles.css` still line up).
- When we add a new block type whose class is currently dropped (e.g. if
  we later render collection views, stop dropping `notion-collection*`).
